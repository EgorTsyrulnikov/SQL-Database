# SQL-базы данных: Практическое задание

Данный репозиторий содержит выполненное практическое задание по работе с базами данных с использованием языка SQL. 

## Цели и задачи проекта
Основная цель проекта — закрепление навыков проектирования баз данных, написания SQL-запросов различной сложности, использования агрегации, группировок, подзапросов, объединений (`JOIN`), а также рекурсивных запросов (`WITH RECURSIVE`).

В рамках проекта реализовано создание, наполнение и анализ данных для четырех различных предметных областей.

## Структура репозитория и описание баз данных

Проект разделен на 4 директории (по одной на каждую предметную область):

1. **`01_vehicles/` — Транспортные средства**
   * **Структура БД:** Включает общую таблицу `Vehicle` (Транспортное средство) и связанные таблицы для конкретных типов: `Car` (Автомобили), `Motorcycle` (Мотоциклы) и `Bicycle` (Велосипеды).
   * **Реализованные функции:** Выборка данных по техническим характеристикам (мощность, объем двигателя, цена) с использованием `JOIN` и объединение разнородных результатов с помощью `UNION ALL`.

2. **`02_races/` — Автомобильные гонки**
   * **Структура БД:** Таблицы `Classes` (Классы), `Cars` (Машины), `Races` (Гонки) и таблица связи `Results` (Результаты).
   * **Реализованные функции:** Расчет средних позиций автомобилей и классов, аналитические запросы с использованием `AVG()`, `COUNT()`, `MIN()`, фильтрация и сложная сортировка результатов гонок.

3. **`03_hotels/` — Бронирование отелей**
   * **Структура БД:** Таблицы `Hotel` (Отель), `Room` (Номера), `Customer` (Клиенты) и `Booking` (Бронирования).
   * **Реализованные функции:** Вычисление потраченных клиентами средств (с использованием дат), категоризация отелей по стоимости (Дешевый, Средний, Дорогой) через `CASE`, определение предпочтений клиентов и конкатенация строк (`GROUP_CONCAT`).

4. **`04_organization/` — Структура организации**
   * **Структура БД:** Таблицы иерархии `Employees` (Сотрудники), `Departments` (Отделы), `Roles` (Роли), `Projects` (Проекты) и `Tasks` (Задачи).
   * **Реализованные функции:** Построение иерархии подчинения сотрудников с помощью рекурсивных обобщенных табличных выражений (`WITH RECURSIVE`), расчет общего числа задач и общего числа подчиненных.

В каждой директории находятся файлы:
* `schema.sql` — скрипт DDL (создание таблиц) и DML (наполнение тестовыми данными) для СУБД MySQL.
* `tasks.sql` — SQL-запросы (с комментариями), содержащие решения поставленных задач.

## Инструкция по запуску и тестированию

1. Установите локальный веб-сервер, включающий СУБД MySQL (например, [XAMPP](https://www.apachefriends.org/)).
2. Запустите службы **Apache** и **MySQL** в контрольной панели XAMPP.
3. Откройте интерфейс **phpMyAdmin** (обычно доступен по адресу `http://localhost/phpmyadmin`).
4. Создайте 4 пустые базы данных (например: `db_vehicles`, `db_races`, `db_hotels`, `db_organization`).
5. Для каждой базы данных перейдите на вкладку **SQL**, скопируйте содержимое соответствующего файла `schema.sql` и нажмите **Выполнить**.
6. После успешного создания таблиц и наполнения их данными скопируйте запросы из файлов `tasks.sql` и поочередно выполняйте их для проверки.
7. Результаты выполнения запросов можно сверить с ожидаемыми тестовыми данными, представленными в задании.
